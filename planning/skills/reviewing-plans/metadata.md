# metadata

- **Название:** reviewing-plans
- **Автор:** Константин Коновалов
- **Версия:** 1.0.0
- **Описание:** Критически ревьюит написанный план реализации до старта кодинга: проверяет покрытие спека, плейсхолдеры, технические риски, тесты, порядок зависимостей, консистентность типов/имён, оверинжиниринг, коммиты.
- **Применимость:** Используйте после `writing-plans`, до `executing-plans`. Третье звено цепочки `brainstorming → writing-plans → reviewing-plans → executing-plans`. Можно запускать независимо на любом плане из `docs/plans/`.
- **Использование:** Читает план (последний из `docs/plans/` или по явному пути) и спек (из `docs/specs/`, если есть). Выводит таблицу находок с северити прямо в чат — файл не пишет. При наличии Blocker-ов предлагает исправить план инлайн.
