# metadata

- **Название:** writing-plans
- **Автор:** Jesse Vincent (@obra) / superpowers, адаптация Константин Коновалов
- **Версия:** 1.0.0
- **Описание:** Создаёт пошаговый план реализации из спека: задачи по 2–5 минут, каждая с TDD-шагами (тест → запуск красный → реализация → запуск зелёный → коммит). Документирует какие файлы трогать, как тестировать, что проверить. Сохраняет план в `docs/plans/`.
- **Применимость:** Используйте когда спек одобрен (после `brainstorming`) и нужен конкретный план реализации перед стартом кодинга. Второе звено цепочки `brainstorming → writing-plans → reviewing-plans → executing-plans`.
- **Использование:** Получает спек (файл из `docs/specs/` или описание в запросе), генерирует план и сохраняет в `docs/plans/<YYYY-MM-DD>-<тема>-plan.md`. Затем передаёт управление в `executing-plans`.
