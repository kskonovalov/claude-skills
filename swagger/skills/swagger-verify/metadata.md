# metadata

- **Название:** swagger-verify
- **Автор:** Константин Коновалов
- **Версия:** 1.0.0
- **Описание:** Проверяет корректность уже готового OpenAPI/Swagger описания модуля Node.js — `.yml`, `.json` или inline `@swagger` JSDoc-блоков. Две проверки: формальная валидность OpenAPI 3.0 (синтаксис + схема + `$ref`) и соответствие коду (нет выдуманных и потерянных роутов, путь/метод/параметры/ответы/security/модели совпадают, нет незакрытых `TODO`).
- **Применимость:** Парная проверка к `swagger-yml` / `swagger-json` / `swagger-jsdoc` — запускать после генерации или для аудита чужой спеки. Не для дизайна нового API (→ `api-designer`) и не для ревью контракта во всём проекте (→ агент `code-review-contracts`).
- **Использование:** На входе — **что проверять** (артефакт `.yml`/`.json` или модуль с inline-блоками) и **с чем сверять** (исходный модуль; если не указан — ищет сам, иначе спрашивает одним вопросом). Прогоняет `npx @redocly/cli lint`, сверяет роуты/параметры/ответы/модели с кодом, выдаёт отчёт таблицей **в чат** (🔴 ошибка / 🟡 расхождение / ⚪ заглушка). Файлы и код не правит — только сообщает; исправление = перегенерировать соответствующим swagger-скиллом.
