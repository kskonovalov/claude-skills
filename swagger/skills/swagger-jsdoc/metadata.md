# metadata

- **Название:** swagger-jsdoc
- **Автор:** Константин Коновалов
- **Версия:** 1.0.0
- **Описание:** Встраивает OpenAPI-описание прямо в код модуля Node.js в виде JSDoc-блоков `/** @swagger ... */` над обработчиками роутов (формат библиотеки `swagger-jsdoc`): путь, метод, параметры, requestBody, ответы, security. Переиспользуемые модели выносит в один блок `components.schemas` через `$ref`.
- **Применимость:** Когда проект собирает спеку через `swagger-jsdoc` (сканирует JSDoc по `apis: [...]`) и нужно аннотировать модуль inline, а не отдельным файлом. Для самостоятельного файла — `swagger-yml` / `swagger-json`. Не для дизайна нового API (→ `api-designer`).
- **Использование:** На входе ждёт **путь к модулю**. Находит роуты, собирает параметры/ответы/auth, вставляет блок `@swagger` над каждым объявлением (путь в OpenAPI-формате: `/document/{uuid}`, не `:uuid`), схемы — один раз через `components.schemas`. Логику кода не меняет, только добавляет комментарии; незнакомые типы помечает `# TODO: уточнить`, роуты не выдумывает.
