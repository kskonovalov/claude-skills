# metadata

- **Название:** swagger-yml
- **Автор:** Константин Коновалов
- **Версия:** 1.0.0
- **Описание:** Генерирует отдельный валидный OpenAPI/Swagger 3.0 **`.yml`** файл для одного модуля Node.js (Express/Fastify/Nest), вытаскивая контракт строго из кода: роуты, path/query/body/headers-параметры, коды и тела ответов, модели (ORM/валидаторы/TS-типы) и схемы авторизации.
- **Применимость:** Когда нужен самодостаточный YAML-файл со спекой модуля. Для JSON — `swagger-json`, для inline-аннотаций в коде — `swagger-jsdoc`. Не для дизайна нового API (→ `api-designer`) и не для ревью согласованности контракта (→ агент `code-review-contracts`).
- **Использование:** На входе ждёт **путь к модулю**. Идёт по 8 шагам: входная точка → роуты → параметры → ответы → модели → безопасность → сборка OpenAPI → запись и валидация. Артефакт сохраняется внутри проекта в папку AI-инструмента: `<ai-tool>/swagger/<module-name>.swagger.yml` (напр. `.claude/swagger/users.swagger.yml`), проверка `npx @redocly/cli lint`. Ненайденные типы помечает `# TODO: уточнить`, роуты не выдумывает.
