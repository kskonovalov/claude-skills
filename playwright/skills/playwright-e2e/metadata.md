# metadata

- **Название:** playwright-e2e
- **Автор:** Константин Коновалов
- **Версия:** 1.1.0
- **Описание:** Генерирует один блок `test.describe.only("<что проверяем>", () => { ... })` с E2E-тестами Playwright для React-компонента по пути к файлу. Собирает элементы по тест-атрибутам (в DOM все — `data-test-id`; React-проп `testId` тоже становится `data-test-id`), выводит сценарии из кода (обработчики, состояния, условный рендер, комбинации пропсов), показывает список на подтверждение и пишет тесты через `getByTestId()`. Блок рассчитан на вставку в уже настроенный Playwright-спек.
- **Применимость:** Когда есть React-компонент и нужен стартовый набор E2E для расширения покрытия. Работает по коду, не выдумывает. НЕ для: настройки Playwright с нуля, component-тестов в изоляции (`@playwright/experimental-ct-react`), unit-тестов логики (→ Jest/Vitest), ревью существующих тестов. Смежное: `code-review-*` (ревью), `react-testing` (стратегия).
- **Использование:** На входе — **путь к компоненту/файлу** (обязательный; если нет — спрашивает одним вопросом) и **опционально сценарии словами** (включая «проверь разные комбинации»). Выводит кандидатные сценарии из кода, **показывает список в чат и ждёт подтверждения** (HARD-GATE), затем выдаёт **только** `describe.only`-блок через `getByTestId()`. Допущения (не проверяет и не ищет): на всех элементах есть `data-test-id`, Playwright настроен, навигация в `beforeEach` — поэтому не читает `playwright.config`, не добавляет импорты/шапку, не ищет роуты и не пересказывает файл. Сохраняет по умолчанию в `.gigacode/playwright/<component-name>/<component-name>.spec.ts`.
