# metadata

- **Название:** playwright-e2e
- **Автор:** Константин Коновалов
- **Версия:** 1.0.0
- **Описание:** Генерирует Playwright E2E-спеки (`*.spec.ts`) для React-компонента по пути к файлу. Собирает элементы по тест-атрибутам (`data-test-id` / `data-testid` / `testId`), выводит сценарии из кода (обработчики, состояния, условный рендер, комбинации пропсов), показывает список на подтверждение и пишет тесты через `getByTestId()`. Тесты рассчитаны на копипаст в уже настроенный Playwright.
- **Применимость:** Когда есть React-компонент и нужен стартовый набор E2E для расширения покрытия. Работает по коду, не выдумывает. НЕ для: настройки Playwright с нуля, component-тестов в изоляции (`@playwright/experimental-ct-react`), unit-тестов логики (→ Jest/Vitest), ревью существующих тестов. Смежное: `code-review-*` (ревью), `react-testing` (стратегия).
- **Использование:** На входе — **путь к компоненту/файлу** (обязательный; если нет — спрашивает одним вопросом) и **опционально сценарии словами** (включая «проверь разные комбинации»). Выводит кандидатные сценарии из кода, **показывает список в чат и ждёт подтверждения** (HARD-GATE), затем пишет спеку через `getByTestId()`. Роут выводит best-effort, иначе `// TODO: роут`. Скилл ничего не запускает и не конфигурит — только генерирует файлы. Сохраняет по умолчанию в `.gigacode/playwright/<component-name>/<component-name>.spec.ts`.
