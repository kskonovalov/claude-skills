# metadata

- **Название:** code-review-logic (ревью корректности логики)
- **Автор:** Константин Коновалов
- **Версия:** 1.0.0
- **Описание:** Узкий ревьюер одной линзы — корректности поведения кода. Ищет edge cases (пустые массивы, нулевые значения, off-by-one), разыменование null/undefined без guard, перепутанные условия и операторы, неполные ветвления, потерянные await, forEach с async-колбэком, гонки при read-modify-write и мутацию входных аргументов. Код не редактирует.
- **Применимость:** Используйте при ревью нетривиальной бизнес-логики и условных ветвлений; для проверки async/await-кода на корректный порядок выполнения; при работе с разделяемым состоянием и конкурентными операциями.
- **Использование:** Запустить скилл самостоятельно или через оркестратор `code-reviewer`. Анализирует git diff текущей ветки по умолчанию, либо явный путь/diff-файл. Отчёт сохраняется в `<AGENT_HOME>/reports/<YYYY.MM.DD>/code-review-logic.md`.
