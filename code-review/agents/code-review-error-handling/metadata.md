# metadata

- **Название:** code-review-error-handling (ревью обработки ошибок)
- **Автор:** Константин Коновалов
- **Версия:** 1.0.0
- **Описание:** Узкий ревьюер одной линзы — устойчивости к сбоям. Находит проглоченные ошибки (пустой catch), необработанные Promise rejections, незакрытые ресурсы при ошибке (без finally/dispose), потерю контекста ошибки (без cause), несогласованные форматы ошибок, отсутствие таймаутов/ретраев на I/O и утечку внутренних деталей клиенту. Код не редактирует.
- **Применимость:** Используйте при ревью кода, работающего с внешними сервисами, БД или файлами; для проверки новых try/catch блоков и async-операций; при реализации критических путей, где сбои должны обрабатываться явно и надёжно.
- **Использование:** Запустить скилл самостоятельно или через оркестратор `code-reviewer`. Анализирует git diff текущей ветки по умолчанию, либо явный путь/diff-файл. Отчёт сохраняется в `<AGENT_HOME>/reports/<YYYY.MM.DD>/code-review-error-handling.md`.
