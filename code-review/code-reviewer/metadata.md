# metadata

- **Название:** code-reviewer (оркестратор ревью)
- **Автор:** Константин Коновалов
- **Версия:** 1.0.0
- **Описание:** Оркестратор полного код-ревью. Определяет цель (явный путь, git diff текущей ветки или diff-файл), запускает детерминированный тулинг (tsc, eslint) и все 9 суб-скиллов `code-review-*`, затем собирает сводный отчёт с таблицей вердиктов и топ-приоритетами. Сам код не редактирует.
- **Применимость:** Используйте для полного гранулярного ревью перед мержем PR, когда нужно охватить все аспекты качества за один запуск; для быстрой проверки текущей ветки без аргументов; для ревью конкретного файла или папки по явному пути.
- **Использование:** Запустить скилл без аргументов — проверит git diff текущей ветки. Передать путь (`code-reviewer /src/module`) — проверит указанный файл/папку. Передать подмножество аспектов («только security и types») — запустит только нужные суб-скиллы. Отчёты сохраняются в `<.claude|.cursor|.gigacode>/reports/<YYYY.MM.DD>/`.
