# SHARED-SPEC — общий контракт code-review скиллов

> Справочник для сопровождающего набора. **Этот файл не грузится моделью** — он описывает инварианты,
> которые продублированы (self-contained) внутри каждого `SKILL.md`. Меняешь контракт здесь → синхронно
> правишь во всех скиллах.

## Состав

- Оркестратор: `code-reviewer`
- 9 суб-скиллов: `code-review-{logic,error-handling,validation,security,types,performance,architecture,readability,contracts}`

Каждый суб-скилл проверяет **только свою линзу** и явно отдаёт смежные классы проблем соседу (см. раздел
«Границы» в каждом `SKILL.md`). Цель — отсутствие дублей находок между отчётами.

## Тип скиллов

**Action / report**, не guidance. Скилл: определяет вход → анализирует свой аспект → **пишет отчёт в файл**.
Скилл **review-only**: НЕ редактирует исходный код, НЕ запускает правки, НЕ блокирует мерж.

## Вход (одинаков для всех)

Приоритет определения цели ревью:
1. **Явный путь** в запросе (`/path/to/code` — файл или папка) → ревьюим его.
2. **Файл с git diff** в запросе (`*.txt`/`*.diff` с выводом `git diff`) → ревьюим его содержимое.
3. **По умолчанию** — `git diff` текущей ветки относительно базовой; если базовой ветки/гита нет —
   fallback на незакоммиченные изменения (`git diff HEAD`). Самодостаточный сниппет:
   ```bash
   BASE=$(git merge-base HEAD main 2>/dev/null || git merge-base HEAD master 2>/dev/null)
   [ -n "$BASE" ] && git diff "$BASE"...HEAD || git diff HEAD
   ```

## Путь отчёта (не захардкожен)

1. Определить `AGENT_HOME` = первая существующая в корне проекта папка из списка по порядку:
   `.cursor/`, `.gigacode/`, `.claude/`. Если ни одной нет — создать `.claude/`.
2. Каталог дня: `<AGENT_HOME>/reports/<YYYY.MM.DD>/` (создать при отсутствии).
3. Файл: `<AGENT_HOME>/reports/<YYYY.MM.DD>/<skillname>.md`, где `<skillname>` = имя скилла
   (`code-review-logic`, …, для оркестратора `code-reviewer`).
4. Отчёт пишется **всегда**, даже при вердикте PASS (с пустыми секциями находок).
5. Повторный запуск в тот же день перезаписывает файл (история за день не хранится — by design).

## Язык и приоритет (одинаковы для всех)

- **Язык отчёта** = язык запроса пользователя. Если запрос без естественного языка (скормлен только diff) —
  **русский**.
- **Приоритет:** инструкции в запросе пользователя **перекрывают** инструкции скилла — язык, фокус,
  severity-пороги, объём входа, формат. Дефолты применяются только когда запрос молчит.

## Severity

| Уровень | Значок | Что это |
|---|---|---|
| Blocker | 🔴 | Сломает прод / эксплуатируемо / потеря данных. Нельзя мержить. |
| Major | 🟠 | Реальный баг или серьёзный риск, но не катастрофа. Чинить до мержа. |
| Minor | 🟡 | Мелкий дефект / запах кода. Можно отложить. |
| Info | ⚪ | Замечание/наблюдение, не требует действий. |

## Вердикт

- **❌ FAIL** — есть хотя бы один Blocker.
- **⚠️ WARN** — есть Major и/или Minor, но нет Blocker.
- **✅ PASS** — находок нет (либо только Info).

## Шаблон отчёта

```markdown
# Review: <skillname> — <✅ PASS | ⚠️ WARN | ❌ FAIL>
> вход: <git diff текущей ветки | путь <X> | diff-файл <Y>> · дата: <YYYY-MM-DD HH:MM> · находок: <N>

## 🔴 Blocker (<k>)
- `path/to/file.ts:42` — <в чём проблема>.
  Почему: <последствие/риск>. Фикс: <конкретное действие>.

## 🟠 Major (<k>)
- …

## 🟡 Minor (<k>)
- …

## ⚪ Info (<k>)
- …

## ⭐ Итог
<вердикт + одна строка резюме (что главное починить)>
```

Правила находок:
- Всегда `файл:строка` (для diff — путь и строка из заголовков ханков).
- Триада: **проблема → Почему (последствие) → Фикс (что сделать)**. Без воды.
- Только находки своей линзы. Смежное — молча отдать соседнему скиллу (не дублировать).
- Если уверенности нет — Info с пометкой «проверить вручную», не раздувать severity.

## Структура SKILL.md (единая для всех суб-скиллов)

1. Frontmatter: `name`, `description` (3-е лицо + триггеры). 
2. `## Что проверяет` — линза скилла.
3. `## Границы (что НЕ моё)` — явная передача смежного соседу.
4. `## Чеклист` — конкретные паттерны под стек TS/React/Node(NestJS)/Postgres, в идеале с краткими FAIL→FIX.
5. `## Процесс` — определить вход → проанализировать → сформировать находки.
6. `## Отчёт` — путь + шаблон (из этого файла, продублирован).
7. `## Язык и приоритет` — правило языка и приоритета запроса.
