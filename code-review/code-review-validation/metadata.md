# metadata

- **Название:** code-review-validation (ревью валидации входных данных)
- **Автор:** Константин Коновалов
- **Версия:** 1.0.0
- **Описание:** Узкий ревьюер одной линзы — валидации и границ доверия. Проверяет, что все внешние данные (тело запроса, query/path-параметры, заголовки, файлы, env, CLI-аргументы) проходят схемную валидацию с границами значений, нормализацией и whitelisting. Код не редактирует.
- **Применимость:** Используйте для проверки новых API-эндпоинтов и форм на наличие DTO-схем и ограничений; при добавлении обработки внешних данных (webhooks, очереди, сторонние API); для аудита мест, где `JSON.parse` или данные из БД используются без проверки инвариантов.
- **Использование:** Запустить скилл самостоятельно или через оркестратор `code-reviewer`. Анализирует git diff текущей ветки по умолчанию, либо явный путь/diff-файл. Отчёт сохраняется в `<AGENT_HOME>/reports/<YYYY.MM.DD>/code-review-validation.md`.
