# metadata

- **Название:** code-review-architecture (ревью архитектуры и тестируемости)
- **Автор:** Константин Коновалов
- **Версия:** 1.0.0
- **Описание:** Узкий ревьюер одной линзы — структуры кода и тестируемости. Проверяет соблюдение границ слоёв (бизнес-логика не в контроллере/компоненте), направление зависимостей (домен не зависит от инфраструктуры), SRP, отсутствие god-объектов, циклические зависимости и скрытые зависимости (прямой new, Date.now(), fetch без инъекции), препятствующие юнит-тестированию. Код не редактирует.
- **Применимость:** Используйте при создании новых модулей или сервисов, чтобы проверить правильность слоёв; при рефакторинге для выявления структурного долга; для оценки тестируемости новой логики до написания тестов.
- **Использование:** Запустить скилл самостоятельно или через оркестратор `code-reviewer`. Анализирует git diff текущей ветки по умолчанию, либо явный путь/diff-файл. Отчёт сохраняется в `<AGENT_HOME>/reports/<YYYY.MM.DD>/code-review-architecture.md`.
