# metadata

- **Название:** code-review-readability (ревью читаемости и именования)
- **Автор:** Константин Коновалов
- **Версия:** 1.0.0
- **Описание:** Узкий ревьюер одной линзы — читаемости и именования. Выявляет неинформативные и вводящие в заблуждение имена, слишком длинные функции и глубокую вложенность, магические числа/строки без констант, мёртвый и закомментированный код, устаревшие комментарии и смысловое дублирование. Фокус — на смысловой ясности, не на стиле (prettier/eslint). Код не редактирует.
- **Применимость:** Используйте при ревью сложной бизнес-логики, где важна читаемость для будущих разработчиков; для проверки именования в новых модулях и функциях; при рефакторинге запутанного или плохо структурированного кода.
- **Использование:** Запустить скилл самостоятельно или через оркестратор `code-reviewer`. Анализирует git diff текущей ветки по умолчанию, либо явный путь/diff-файл. Отчёт сохраняется в `<AGENT_HOME>/reports/<YYYY.MM.DD>/code-review-readability.md`. Самая «мягкая» линза — severity обычно Minor/Info.
