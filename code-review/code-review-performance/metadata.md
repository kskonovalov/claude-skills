# metadata

- **Название:** code-review-performance (ревью производительности)
- **Автор:** Константин Коновалов
- **Версия:** 1.0.0
- **Описание:** Узкий ревьюер одной линзы — производительности под стек React + Node.js + PostgreSQL. Находит N+1 запросы, отсутствующие индексы, лишние ре-рендеры React (нестабильные пропсы, отсутствие memo), O(n²) алгоритмы, блокирующий синхронный I/O, последовательные await вместо Promise.all, отсутствие пагинации и стриминга. Код не редактирует.
- **Применимость:** Используйте при добавлении новых запросов к БД или циклов с I/O; для проверки React-компонентов на лишние ре-рендеры; при ревью кода, работающего с большими коллекциями данных или в горячих путях под нагрузкой.
- **Использование:** Запустить скилл самостоятельно или через оркестратор `code-reviewer`. Анализирует git diff текущей ветки по умолчанию, либо явный путь/diff-файл. Отчёт сохраняется в `<AGENT_HOME>/reports/<YYYY.MM.DD>/code-review-performance.md`.
